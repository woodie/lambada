# lambada 2.4.0

## Developer tooling: JS linting and testing

`cmd/lambada-web`'s inline delete-confirmation script is now its own file,
`cmd/lambada-web/static/script.js`, served via `go:embed` plus a new
`GET /script.js` route -- mirroring scandalous's `public/script.js`. It has
real linting and testing for the first time:
[`standard`](https://standardjs.com) and [`vitest`](https://vitest.dev),
with a `context = describe` alias so specs read like the RSpec-style
hierarchy used elsewhere in this account.

## CI now also runs golangci-lint and the JS suite

`.github/workflows/ci.yml` is new for this repo -- a `golangci-lint` job
(default linter set, no `.golangci.yml`) and a `javascript` job.

## npm scripts for the whole dev loop, plus LAMBADA_QUIET

`lint-js`/`test-js`/`lint-go`/`test-go` cover each half individually;
`npm run check` runs all four in sequence and stops at the first failure.
The Go side of `check` sets `LAMBADA_QUIET=1`, a new env var both
binaries' `init()` honor to silence all `log.Printf`/`log.Fatalf` output,
so `ginkgo -r`'s pass/fail dots aren't buried in every handler's log
lines -- see `docs/DEVELOPMENT.md` for the full rundown.

## Internal rename: templates/ -> views/, Scans -> Listing

`cmd/lambada-web/templates/listing.html.tmpl` moved to
`cmd/lambada-web/views/`, and `listingData`'s `Scans` field is now
`Listing`. Internal only -- doesn't affect `/files.json`'s wire format or
any other public behavior.

No other user-facing changes.
