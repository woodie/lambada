# lambada 2.8.0

## Issue #17 closed out: status codes visible at the route, no hidden helper

`cmd/lambada-web/main.go`'s 404/500 responses now go through `http.Error`
directly at each route -- matching the one-line stdlib idiom already used
for the 500 case, instead of a hand-rolled `w.WriteHeader` + `w.Write`
pair. `scanFilesListingOrFail` (a wrapper that mixed "fetch the listing"
with "write a 500 on failure") is gone; the index and `/files.json` routes
call `scanFilesListing` directly and write their own 500 inline, the same
shape the 404 handling in download/delete already had. `scanfiles.go` no
longer imports `net/http` or `log` at all -- it's pure filesystem work now,
matching `scandalous/lib/scan_files.rb`'s equivalent boundary.

## `LAMBADA_QUIET` replaced by `LOG_LEVEL=OFF`

`loglevel` is a new, plain (non-`internal/`) package: `loglevel.Apply()`
checks `LOG_LEVEL == "OFF"` and discards the standard logger's output.
`LOG_LEVEL` is a widely-used convention across other languages' tooling,
unlike the old app-specific `LAMBADA_QUIET`. Both binaries' `init()` (moved
to sit directly above `main()` in each file) call it. `package.json`'s
`check` script sets `LOG_LEVEL=OFF` instead of `LAMBADA_QUIET=1` for the
`ginkgo -r` step.

`envOr` (the env-var-with-fallback helper, previously duplicated between
`cmd/lambada-web` and `cmd/lambada-mta`) is now `envutil.Or`, shared from
`internal/envutil` -- one copy instead of two, no version pin needed since
it's the same module.

## humane bumped to v0.9.4: `TimeAgo` takes a value, not a pointer

`humane.TimeAgo` now takes `at time.Time` directly instead of
`at *time.Time` -- every real caller's timestamp is a real, never-nil
value, so the pointer only ever existed to be taken and immediately
dereferenced. `lambada`'s own `timeAgo` adapter (which existed solely to
supply `&t`) is gone; `listingTemplate`'s `FuncMap` references
`humane.TimeAgo` directly, the same way it already did for
`humane.HumanSize`. `humane.DistanceInTime` is unchanged -- still
`*time.Time`, still `WhenNil`-aware -- see `humane`'s own
`docs/releases/v0.9.4.md`.

## Small Go-idiom fixes

`scanFilesListing`'s sort now uses `slices.SortFunc` instead of
`sort.Slice` (`go.mod` already targets `go 1.26.3`). Its `Glob` error is
now wrapped with `%w` and the directory path instead of returned bare.
`cmd/lambada-mta/attachments.go`'s multipart-reading loop now checks
`errors.Is(err, io.EOF)` instead of `err == io.EOF`.

## html-validate added for the Go templates

Both `lambada` and `scandalous` picked up `html-validate` this session
(`npm run lint-html`, folded into `check`) since `herb` only understands
Ruby's `<% %>` delimiters, not Go's `{{ }}`. Along the way, a latent
double-quote-nesting bug in the delete button's `onclick` attribute (present
in both templates) was fixed by switching the outer attribute to single
quotes, and both templates gained a missing `lang="en"` on `<html>`.

## Three CI fixes

`golangci-lint-action` bumped to `@v9` (`@v6` only tracks golangci-lint's
v1 line, which stopped following current Go versions). The stale
GitHub-scaffolded `.github/workflows/go.yml` (the only place `go test`
actually ran) was folded into `ci.yml` as a `go-test` job, then deleted --
one workflow per push instead of two. `vitest` bumped to `^4.1.10` to
resolve an `ERESOLVE` conflict with `html-validate`'s optional peer
dependency, now that neither repo commits a `package-lock.json`.

## Upgrading

`go.mod`'s `github.com/woodie/humane` requirement bumped to `v0.9.4`. Run
`go get github.com/woodie/humane@v0.9.4 && go mod tidy` to regenerate
`go.sum`.

If you set `LAMBADA_QUIET=1` anywhere (a systemd override, a shell
profile), switch it to `LOG_LEVEL=OFF` -- the old var is no longer read.

No other user-facing changes; `/files.json`'s wire format is untouched.
