# lambada 2.5.0

## Listing now shows "about" for hour-plus timestamps

`lambada-web`'s `timeAgo` opts into `humane`'s new `Approximate: true` option
(`humane` `v0.4.0`): timestamps an hour or older now render as `"about 15 hours
ago"`, `"about 1 day ago"`, etc., instead of an exact-looking count. Anything
under an hour (`"3 minutes ago"`, `"less than a minute ago"`) is unchanged.

This listing is a static, server-rendered page with no way to refresh itself
once loaded -- exact hour/day counts overstate their own precision the moment
the page sits open for a while. `Approximate` exists specifically for that
case; see `humane`'s own `docs/releases/v0.4.0.md`. `scandalous`'s `time_ago`
opted into the identical option in its own `2.5.0`.

## Upgrading

`go.mod`'s `github.com/woodie/humane` requirement bumped to `v0.4.0`. Run
`go get github.com/woodie/humane@v0.4.0 && go mod tidy` to regenerate `go.sum`.

No other user-facing changes.
