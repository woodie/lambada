# lambada 2.7.0

## Adopting humane v0.9.0's full API rethink

`humane` dropped its instantiated-formatter shape entirely in `v0.9.0` --
`humane.SizeFormatter{}`/`humane.TimeFormatter{Approximate: true}` and
their `.Format` methods are gone, replaced by package-level
`humane.HumanSize`/`humane.TimeAgo` functions (see `humane`'s own
`docs/COWORK.md` `v0.9.0` entry for the full cross-repo rationale).
`listingTemplate`'s `FuncMap` now references `humane.HumanSize` directly
and wraps `humane.TimeAgo(&t, time.Now())` inline; the module-level
`sizeFormatter`/`timeFormatter` vars are gone with them -- there was no
per-instance state left to hold once configuration moved to a per-call
argument.

`humane.TimeAgo`'s `Approximate` now defaults to `true` (was `false`,
requiring the explicit `TimeFormatter{Approximate: true}` this file used
to set) -- matches what this app already opted into, so the rendered
listing is unchanged. `main_test.go`'s existing assertions (`"less than a
minute ago"`, `"in 3 minutes"`, the `"80 KB"` fixture) all stay under the
hour-scale `Approximate` boundary or are otherwise untouched by
`HumanSize`'s rounding correction, so nothing needed updating there.

Confirmed live on the Pi: rebuilt and redeployed via `go mod tidy` +
`make install` + `systemctl restart`, then re-verified against a real
file (`"226 KB"`, `"1 minute ago"` on the listing page).

## Upgrading

`go.mod`'s `github.com/woodie/humane` requirement bumped to `v0.9.0`. Run
`go mod tidy` to regenerate `go.sum` against the real published module.

No other user-facing changes.
