# lambada 2.6.0

## Listing now matches ActionView's bucket table exactly

`lambada-web`'s `timeAgo` picks up `humane v0.5.0` (see `humane`'s own
`docs/releases/v0.5.0.md`), which reworked `TimeFormatter` to match
ActionView's `distance_of_time_in_words` bucket table exactly, through the
"1 day" row. Two output changes on the listing page as a result:

- Timestamps under a minute now collapse to `"less than a minute ago"` only
  under 30 seconds, not 60 -- a 45-second-old file now reads `"1 minute ago"`.
- `Approximate`-prefixed "about" wording no longer applies to the day bucket:
  a 30-hour-old file now reads `"1 day ago"`, not `"about 1 day ago"`. Anything
  from 45 minutes to 24 hours old still reads `"about ..."`, unchanged, and now
  kicks in a little earlier too (44 minutes 30 seconds, not a full hour).

Confirmed live on the Pi: rebuilt and redeployed via `make install` +
`systemctl restart`, then re-verified against real files with adjusted
`touch` timestamps at each of the table's boundaries.

## Upgrading

`go.mod`'s `github.com/woodie/humane` requirement bumped to `v0.5.0`. Run
`go get github.com/woodie/humane@v0.5.0 && go mod tidy` to regenerate `go.sum`.

Test suite also dropped the boundary-value `timeAgo` cases in
`cmd/lambada-web/main_test.go` that duplicated `humane`'s own now-comprehensive
coverage of this table -- kept the size-listing assertion, the "just now"
wiring check, the future-direction case (tied to a real historical
regression, issue #15), and the delete-confirm-dialog wiring check.

No other user-facing changes.
