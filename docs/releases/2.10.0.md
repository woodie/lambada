# lambada 2.10.0

## Test suite migrated off Ginkgo/Gomega onto spec + expect

No app behavior changed in this release -- this is entirely about how `cmd/lambada-web` and `cmd/lambada-mta` are tested. Worth its own release notes anyway: `lambada` is treated as a sample app for this account's Go conventions, so a change to how it's tested is exactly the kind of thing worth writing down here, not just in `docs/COWORK.md`.

Both binaries' test suites dropped `onsi/ginkgo`/`onsi/gomega` entirely, in favor of two small libraries built alongside this migration:

- [`github.com/woodie/spec`](https://github.com/woodie/spec) -- a fork of `sclevine/spec` (a BDD test organizer that runs on real `go test` subtests, no framework of its own) extended with a handful of additions Go's evolution since 2019 now makes possible: `it.Context()`/`it.T()` (pass-through access to the real `t.Context()`/`*testing.T`), `Describe`/`AsContext()` (RSpec-style `describe`/`context` duality), `Var[T]` (a generics-based typed box for sharing state between a `Before` and its `it`s), and `RunAliased` (passes `describe`/`context`/`it`/`before`/`after` in as plain callback parameters, no per-file alias line needed anywhere).
- [`github.com/woodie/expect`](https://github.com/woodie/expect) -- a small, generics-based matcher library, built new this cycle specifically to replace Gomega without adopting it: a matcher can be statically typed to what it actually compares instead of leaning on `interface{}` and reflection, and an assertion just takes `*testing.T` directly instead of a global fail handler. Dot-imported, so call sites read almost identically to their old Gomega equivalents (`Expect(t, x).To(Equal(y))`).

Every migrated test file now reads as:

```go
spec.RunAliased(t, "Lambada WEB", func(t *testing.T, describe, context spec.Describe, it spec.S, before, after func(func())) {
    describe("GET index", func() {
        context("with no files", func() {
            it("renders the empty state", func() {
                rec := get(mux, "/")
                Expect(t, rec.Code).To(Equal(http.StatusOK))
            })
        })
    })
})
```

Both `spec` and `expect` are consumed as real, tagged, published modules (`v0.1.0`/`v0.1.1`), not local checkouts -- `lambada` was the real end-to-end test case for both, confirmed by `go mod tidy` fetching each over the network and the full suite (55 specs across both binaries) passing clean.

## Fixed: a few `describe` names were confusing `gorderly`'s tree render

`spec`'s default mode joins nested names with `/`, and `gorderly` rebuilds its tree by splitting on `/` -- so `main_test.go`'s HTTP-path-shaped names (`"GET /"`, `"GET /download/{filename}"`, and similar) had their own slashes mistaken for real hierarchy. Renamed to `"GET index"`, `"GET download by filename"`, etc. -- labels only, no route or behavior changed.

## Tooling

`package.json`'s `test`/`test-go` scripts changed from `ginkgo -r`/`LOG_QUIET=1 ginkgo -r` to `go test -v ./... | gorderly -fd`/`LOG_QUIET=1 go test ./...` -- the `ginkgo` CLI is no longer a dependency of this repo at all.

## Upgrading

Nothing to do for anyone deploying or running `lambada` -- no wire-format, config, or runtime behavior changed. If you build from source and previously needed the `ginkgo` CLI installed to run tests, you don't anymore; `go test ./...` (or `npm run test`/`npm run check`) is all `make check`/CI ever needs now.
