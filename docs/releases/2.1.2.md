# lambada 2.1.2

## Bug fix: file sizes now match Finder

2.1.1 fixed lambada-web's file-size label ("kB" → "KB") but not the underlying math — it moved to 1024-based division, so a real file's size disagreed with what Finder (and the zouk app) report for the same file, e.g. "220.6 KB" vs Finder's "226 KB". No published Go library ships 1000-based math under capitalized "KB"/"MB" labels, so `humanSize` is now a small hand-rolled function (no external dependency) using the same rounding approach `go-humanize` always used, just with the right base and capitalization. File sizes on the listing page now match what Finder and zouk report for the same file.

No other user-facing changes.
